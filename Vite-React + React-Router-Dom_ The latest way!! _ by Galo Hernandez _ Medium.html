Vite-React + React-Router-Dom: The latest way!!

Galo Hernandez
8 min read
·
Feb 19, 2023

--

The short story is that I was following just another e-commerce tutorial and found out that most of them are OUTDATED so I challenged myself to find how to resolve the puzzle with the latest versions of everything.

I must mention that the tutorial I am using is from 2021 so it seems that is very, very old now by today's standards. Those kinds of things are the obstacles most web developers face day after day.

This will be the first of several mini-tutorials. Hopefully, I will share with you my advances in the easiest way to follow and someday finish the complete e-commerce web application.

In this mini-tutorial, we will end with a react-router-dom implementation with 3 pages using Vite+React and for styling Antd and Bootstrap.

Hopefully, you will follow these steps to get the same result:

  1. React installation

First things first! What is the proper way to install React? I will not go deep on this but it seems most “YouTube web gurus” say that “create-react-app” is finished and most of them recommend using Vite to do this so I accepted the challenge.

I had several problems with “create-react-app” too. For example, the “npm start” does not work properly because I use the latest Node and NPM versions so I had to use some hacks to make this work. I can share that hack with you in the messages section if you want.

By the way, my oldie e-commerce tutorial uses “create-react-app”.

I will use VS Code throughout all these series of mini tutorials.

Just because we are creating an e-commerce app I named the front-end part “client”. For installing React with Vite open your terminal and use this command:

npm create vite@latest client -- --template react

The process is very fast compared to “create-react-app”. After finishing it will ask you to cd to your client folder and then run the command:

npm install

It is possible that you will have to fix some warnings. You can do this if you want with “npm audit-fix”.

Then you can run the project using this line of code:

npm run dev

The console will tell you that your app is running in localhost: 5173 (this is already different from the usual 3000 port!).

Then you will see the Vite+React basic front page:

When you look at the structure tree you can notice some differences from the “usual” React installation:

  • There is no index.js but main.jsx
  • no vitals js
  • a vite-config.js
  • a vite svg (for the basic front)
  • other stuff in node_modules that are beyond the purpose of this mini-tutorial

2. The cleaning

The cleaning process is for just getting rid of any code that we will not actually use.

I remove everything from App.jsx. It will end like this:

function App() {

  return (
    <div className="App">
      Vite APP
    </div>
  )
}

export default App

You can delete:

  • app.css
  • the svg
  • just lets index.css without any code

3. Back to the tutorial guide!

As I mentioned before I am following an oldie 2021 e-commerce tutorial. The author wants me to use 2 different styling solutions:

  • Antd: This is an implementation for React of the Ant Design principles. Well, that is what they say on their website. I don't know why some modern websites don't have a little history or “about this” link. Anyway, to install this you need to run this line of code:
npm install antd
  • Bootstrap — React-bootstrap: I guess we all know what is Bootstrap but I will resume that as a collection of design components and other stuff. In 2023 is not recommended by most “web gurus” too. Tailwind is the new thing but we will follow our original instructions. To install run this line of code:
npm install react-bootstrap bootstrap

and in the main.jsx import it:

import 'bootstrap/dist/css/bootstrap.min.css';

And there you are ready to go for using both CSS libraries

5. About installing and using the new react-router-dom v.6

This is kinda the “tricky” part of this mini-tutorial because the documentation for our purpose is not very clear for the latest version of react-router-dom. (Version 6).

To install the latest version run this line of code in your terminal:

npm install react-router-dom localforage match-sorter sort-by

In the 5 version and past versions, we were used to encapsulating our pages inside our Layout:

import React from "react";
import { BrowserRouter as Router, Route } from "react-router-dom";
import Layout from "./Layout";
import HomePage from "./HomePage";
import AboutPage from "./AboutPage";

const App = () => {
  return (
    <Router>
      <Layout>
        <Route exact path="/" render={() => <HomePage />} />
        <Route exact path="/about" render={() => <AboutPage />} />
      </Layout>
    </Router>
  );
};

export default App;

now with the new version, the Layout is included in an “element” property of our root path that will encapsulate all pages but uses some “special components” that are kinda hard to find in the new documentation.

I tried to find a solution in the react-router-dom documentation but they only include a few examples in different Layout situations.

After failing to find answers there I tried to find some solutions in Google, then StackOverflow and others but no luck.

Finally, I went to the new dev toy CHATGPT but the solutions it gave me were outdated too, and after a few “apologies” from HIM/HER, I tried to return to the documentation and look for a solution.

import React from "react";
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Layout from "./Layout";
import HomePage from "./HomePage";
import AboutPage from "./AboutPage";

const App = () => {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Layout />}>
          <Route path="/" element={<HomePage />} />
          <Route path="/about" element={<AboutPage />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
};

export default App;

This last code was generated by ChatGPT but is actually wrong because BrowserRouter is not used anymore in the latest version of react-router-dom

So, I returned back to react-router-dom documentation and after watching some YouTube things here and other things on Google I managed to figure out how to implement this the proper way.

Getting back to our project we have to implement this for our case:

We need to use some functionalities from react-router-dom version 6 :

  • createBrowserRouter
  • createRoutesFromElements
  • RouteProvider
  • Route

Now use them in your App.jsx file:

import React from 'react';
import { Route, createBrowserRouter, createRoutesFromElements, RouterProvider } from 'react-router-dom';
import Home from './pages/Home';
import Login from './pages/auth/Login';
import Register from './pages/auth/Register';
import Header from './components/nav/Header';



const router = createBrowserRouter(
  createRoutesFromElements(
    <Route path="/" element={<Header />}>
      <Route index element={<Home />} />
      <Route path="login" element={<Login />} />
      <Route path="register" element={<Register />} />
    </Route>
  )
)

function App({routes}) {

  return (
    <>
      <RouterProvider router={router}/>
    </>
  );
}

export default App;

The Header element is used as a property of the root path “/” but in order to make it work in all paths including the “Home component that is /” you have to define an “index element” for the Home component.

We now have to adapt the Header component to fit our project needs and use “react-router-dom”.

We will use these elements from the react-router-dom version 6:

  • Outlet
  • Link

After making some changes like deleting some code parts related to submenus and changing icons we will get a code something like this:

import { HomeTwoTone, EditTwoTone, CheckCircleTwoTone } from '@ant-design/icons';
import { Menu } from 'antd';
import { useState } from 'react';
import { Outlet, Link } from 'react-router-dom';


const Header = () => {
  const [current, setCurrent] = useState('h');
  const onClick = (e) => {
    console.log('click ', e);
    setCurrent(e.key);
  };
  return (
    <>
     <Menu onClick={onClick} selectedKeys={[current]} mode="horizontal">
      <Menu.Item key="h" icon= {<HomeTwoTone />}>
       <Link to="/">Home</Link>
      </Menu.Item>
      <Menu.Item key="r" icon= {<EditTwoTone />}>
        <Link to="/register">Register</Link>
      </Menu.Item>
      <Menu.Item key="l" icon= {<CheckCircleTwoTone />}>
        <Link to="/login">Login</Link>
      </Menu.Item>
     </Menu>
     <Outlet/>
    </>
   
  )
};
export default Header;

Some of the changes we made:

  • We use Outlet to show the contents of the different links.
  • We use Link to replace href and similar
  • We add some icons related to our links nature. For this and more icons, you can visit the Ant Design icons library

Finally, you will get your new Nav built with the latest react-router-dom, latest react, using vite and Antd Design.

But wait me a second… Why I installed Bootstrap and never use it? Well, in my oldie tutorial, the author wants to move to the right just “Register” and “Login” Links.

For this purpose, you can use a class called “ml-auto” but unfortunately, this class is incompatible now with Ant Design components. This doesn't mean that Bootstrap is incompatible with And Design at all. You can use Bootstrap in any component of the application.

Instead, you can use an inline style to make the change. Our header will have this final code in the Menu component:

 <Menu onClick={onClick} selectedKeys={[current]} mode="horizontal">
      <Menu.Item key="h" icon= {<HomeTwoTone />}>
       <Link to="/">Home</Link>
      </Menu.Item>
      <Menu.Item key="r" icon= {<EditTwoTone />} style={{ marginLeft: 'auto' }}>
        <Link to="/register">Register</Link>
      </Menu.Item>
      <Menu.Item key="l" icon= {<CheckCircleTwoTone />} >
        <Link to="/login">Login</Link>
      </Menu.Item>
     </Menu>
